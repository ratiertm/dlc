Sections in order:
1. Title: `# {STAGE} Stage Adapter`
2. Description: 1-line what this adapter does
3. `## When This Runs` - trigger keywords, state.json stage number, SKILL.md Read directive
4. `## Prerequisites` - .lifecycle/ exists, state.json + manifest.json exist, previous gate passes
5. `## Step 0: Stage Initialization` - read state.json, check gate N-1_to_N, update state.json
6. `## Step 1-N: {stage-specific steps}` - varies per stage
7. `## Step {last}: Completion` - register in manifest.json, set completed_at, update state.json, write history, regenerate LIVING-STATE.md, announce
8. `## Anti-Patterns` - table with #, Anti-Pattern, Why It Is Wrong
9. `## Relationship to Other Skills` - primary/supporting skill descriptions
10. `## File Locations` - table with File, Path, Purpose

Key patterns from commit-stage.md Step 0:
```markdown
1. Read `.lifecycle/state.json` and verify:
   - `current.feature` has a value
   - `current.stage` = `{N}` (or transition from {N-1} to {N})

2. Check gate condition ({N-1}_to_{N}):
   - Read `.lifecycle/manifest.json`
   - Verify: `artifacts.{N-1}_{prev}.outputs.length > 0`
   - If gate fails: set `current.status = "blocked"`, display missing artifacts, STOP

3. Update `state.json`:
   - `current.stage` = `{N}`
   - `current.stage_name` = `"{STAGE}"`
   - `current.status` = `"in_progress"`
   - `progress.current_stage_started_at` = current ISO 8601 timestamp
```

Completion pattern:
```markdown
1. Register outputs in `manifest.json` under `artifacts.{N}_{stage}.outputs`
2. Set `artifacts.{N}_{stage}.completed_at` = current ISO 8601 timestamp
3. Update `state.json`: status = "completed", session.last_active, session.resume_hint
4. Write history entry to `.lifecycle/history/`
5. Regenerate `.lifecycle/LIVING-STATE.md`
6. Announce completion
```
